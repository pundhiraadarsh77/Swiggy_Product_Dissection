# 🍽️ Swiggy Product Dissection & Database Design

A comprehensive case study of **Swiggy**, one of India's leading hyperlocal commerce platforms. This project analyzes Swiggy's business model, key product features, real-world problems it solves, and the database schema that powers its operations.

---

## 📌 Project Overview

Swiggy has transformed food delivery and quick commerce in India by connecting customers, restaurants, and delivery partners through a technology-driven platform.

This project dissects the product from both **business** and **database design** perspectives, demonstrating how real-world business requirements can be translated into a structured relational database.

---

## 🎯 Objectives

- Understand Swiggy's business model and ecosystem
- Identify real-world problems addressed by the platform
- Analyze key product features
- Design a relational database schema
- Model entity relationships using an ER Diagram
- Demonstrate database design principles

---

## 🗄️ Database Schema

The proposed database consists of the following entities:

- Users
- Restaurants
- Menu Items
- Orders
- Order Items
- Delivery Partners
- Deliveries
- Payments
- Reviews

The schema is designed to efficiently manage customer orders, restaurant operations, deliveries, payments, and customer feedback.

---

## 🔗 Entity Relationships

Some of the key relationships include:

- One User → Many Orders
- One Restaurant → Many Menu Items
- One Order → Many Order Items
- One Delivery Partner → Many Deliveries
- One User → Many Reviews
- One Order → One Payment
